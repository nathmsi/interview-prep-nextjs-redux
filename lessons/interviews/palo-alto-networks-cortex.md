## La boîte — Palo Alto Networks

Palo Alto Networks (NASDAQ: PANW) est la plus grande entreprise **pure-play cybersécurité** au monde par le chiffre d'affaires. Fondée en 2005, réputée pour ses firewalls nouvelle génération, elle s'est étendue via acquisitions vers :

- **Strata** — sécurité réseau (firewalls, SASE, Prisma Access)
- **Prisma** — sécurité cloud (CNAPP, posture, containers, IaC)
- **Cortex** — **Security Operations** (SOC, détection, réponse) ← ton poste

**Cortex** est une plateforme cloud unifiée, AI-driven, pour les équipes SOC. Elle permet d'analyser d'énormes volumes de données de sécurité et d'agir rapidement.

Produits clés sous Cortex :

- **XSIAM** — plateforme SOC nouvelle génération (SIEM + XDR + SOAR + ASM unifiés)
- **XDR** — détection et réponse étendues (endpoints, réseau, cloud)
- **XSOAR** — orchestration et automatisation des réponses
- **Xpanse** — gestion de la surface d'attaque
- **AgentiX** — agents IA pour automatiser le travail SOC

**Contexte 2025 :** stratégie **platformization** — consolidation des produits sur une plateforme unique. Beaucoup d'intégration post-acquisition. Bureau **Tel Aviv** très influent (Twistlock, Demisto, etc.). Culture technique directe.

### Exemple de feature significative (STAR — AI dans l'éditeur)

**Situation :** permettre aux utilisateurs de modifier le design via un chat AI. **Tâche :** connecter les APIs internes de l'éditeur au flux chat. **Action :** intégration APIs, états loading/error, cas limites avec product. **Résultat :** feature en prod, gain de temps mesurable pour designers pro et débutants, contribution à la stratégie AI produit.

### Comment tu gères une app React à millions d'utilisateurs ?

**Perf :** éviter re-renders inutiles, mémoïsation justifiée, lazy loading. **Fiabilité :** error boundaries, loading/error states, tests Jest + e2e. **Maintenabilité :** composants modulaires, TypeScript strict, code reviews. **Déploiement :** feature flags, rollout progressif. Priorité à la **stabilité**.

### Comment tu structures une grosse app React ?

Par **features/domaines**. State **local par défaut**, Context ou Redux pour le global. **Hooks custom** pour logique réutilisable. Types partagés TypeScript. Tests composant + e2e sur parcours critiques. Lisibilité pour 6–15 ingénieurs.
